# METHOD:這套 kit 是怎麼被造出來的(方法論,含真實證據鏈)

> 本文不講 kit 怎麼用(見 README),講**造出它的方法**——因為方法本身可轉移:
> 任何「用流程強制力補齊能力差距」的工程,都適用同一個迴路。
> 所有案例都真實發生於 2026-07-06/07 的開發過程,可在 git log 與 .fable/ 留痕中覆核。

## 迴路:模糊目標 → 精確驗收 → 對抗 → 實跑 → 入法

```
目標(開放詞)──> 驗收(精確詞)──> 實作 ──> 跨模型對抗審查 ──> 金絲雀實跑
                                      ↑                            │
                                      └── Retro/KIT-SUGGESTIONS ←──┘
                                          (人審後入法,規則檔留痕)
```

### 原則一:目標層用開放詞,驗收層用精確詞

「讓機制更穩定」是合法的目標(開放),但動工前必須先變成可判定的驗收:
「紀錄員 throw 時 workflow 必須活著走到 done,且留下失敗 log」——這句話能寫成測試,前一句不能。
反向也成立:驗收精確之後,解空間要留給執行者,不要規定怎麼做。

### 原則二:設計者不得自己出題自評

本 kit 的 v1 自評 95 分,被使用者質疑後交 GPT 側審計推翻(rubric 循環論證、答案預埋)。
此後所有重大結論一律跨模型驗證,且**審查者的修法也要再被審**——本專案多次出現
「修 A 引入 B」被第二輪審查抓到(例:-WhatIf 豁免被 -WhatIf:$false 繞過)。

### 原則三:對抗要有接地驗收,否則比沒有更危險

半壞的橋接層會被 schema 逼著「編造格式正確的空審查」——虛假背書比無審查更糟。
實證:Agy 橋接兩次空轉回泛用句;修法是 echo 接地(審查者必須先複述題目,複述不出即作廢)
+ 結構欄位的 JS 驗收。教訓入法為 REVIEW schema 的 echo 必填欄位。

### 原則四:契約錯位是主要故障類,靜態測試鎖已知、金絲雀抓未知

本專案全部嚴重故障屬同一類——跨層契約只有宣告、沒有驗證:

| 故障(真實案例) | 契約錯位在哪 | 入法後的防護 |
|---|---|---|
| 紀錄員 throw 滅掉 49 分鐘完整 run | falsy 防護 ≠ throw 防護 | try/catch + 契約測試失敗注入 |
| 跨模型審查整份被誤棄 | schema 說 optional、JS 驗證硬要求 | 缺省推導 + 回歸鎖 |
| 監督器誤判截斷、白燒兩次續跑 | 中文狀態行被 haiku 翻成英文標頭 | 格式鐵則 + regex 後備 + 狀態單源 JSON |
| 安全閥誤觸短路(110 秒白燒) | exact-equality 對上模型的「無。<解釋>」 | 語意判定 + 回歸鎖 |

注意最後兩條是**金絲雀實跑**抓到的——stub 測試只能測想像得到的失敗,
「haiku 真的會幻覺/真的會翻譯」只有實跑才現形。兩層互證,缺一不可。

### 原則五:留痕不是官僚,是除錯管道

「降級必須誠實留痕」看似紀律條款,實際上是 approved 誤棄 bug 被發現的原因:
最終回報裡的 `mode: same-family-panel` 與事實(CLI 明明接地成功)矛盾,矛盾即線索。
同理,Retro 迴路第一次實跑就回報了 workflow 自己的 process_mismatch bug——
系統自己報告自己的病,前提是病歷有人寫。

### 原則六:宣稱要嘛附數據,要嘛標注為工作假設

為省 token 砍掉實驗敘事後,外部審查者立刻判「宣稱可信度 4/10」——對讀者而言,
查證不了的實驗等於不存在。修法不是把敘事加回來,是加**證據等級聲明**
(README 首屏:內部消融實驗、數據未公開、採用者視為工作假設)。
benchmark/(對照基準測試)是把工作假設升級為測量的機制。

## 一天的實例:2026-07-06 的完整迴路

init 發佈 → 跨模型審查抓 24 條 → 金絲雀首航(抓 AllowedTools 缺 python、fixture 設計缺陷)
→ 二航(全管線通、Retro 自產 2 條建議、再抓狀態行翻譯漂移)→ 六條 bug 全部入法 + 回歸鎖
→ 三模型家族穩定性分析(共識:靜態防護已達邊際遞減,增量在運行時證據)。
每一步的證據:git log `7ded437..dd5d3a0`、`.fable/KIT-SUGGESTIONS.md`([accepted] 條目)、
契約測試的回歸鎖命名(直接引用故障日期)。

## 把方法帶去別的專案

最小可轉移集:(1) 驗收先於實作;(2) 重大結論跨模型對抗 + echo 接地;
(3) 每個跨層交接物列一條契約測試;(4) 便宜的端到端金絲雀腳本化;
(5) Retro 建議走「[pending] → 人審 → 入法 → 留痕」,不散在對話裡。
