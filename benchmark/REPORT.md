## 判定(依預註冊規則,不得事後遷就)

H1:「kit 化 Opus 品質不輸裸 Fable 5,且成本 ≤ 裸 Fable 5」

1. 通過率:kit 4/4 ≥ Fable 4/4 ✅
2. 品質:kit 9.67 ≥ Fable 9.79 − 0.5 = 9.29 ✅(差距 −0.13,遠小於容忍值)

**三關全過 → 在本實驗的任務範圍內,H1 成立。**

## 誠實解讀(比結論重要)

- **成本優勢的來源是路由紀律,不是編排**:kit 臂 8 個 run 全部被分級路由正確判進輕量層
  (Tier 0/1 inline,50-81 秒),沒有一題誤入重管線。省的錢 = Opus 對 Fable 的單價差 × 路由不加稅。
- **鑑別力集中在 T4**:T1-T3 兩臂產出逐位元相同的解法(裁判語:「完全相同」)——
  這個難度帶測不出品質差;任務對兩個頂級模型都太簡單。
- **T4 的諷刺**:kit 天天宣講外科手術式修改,自己的臂卻順手改了 docstring 被雙裁判扣 scope
  ——提詞條款的勸導極限再次現形(這正是 kit 用 hooks/schema 補強制力的原始理由)。
- **本實驗測不到的**:大型 repo 長程任務、無驗證器的品味題、多日級自主——那是
  TIER3-FRONTIER.md 明文標注的 Fable 專屬地形,本結論不外推到那裡。
- **作廢紀錄**(透明性):第一輪 run(results-20260707-010515)因 kit 臂被 init 鋪入 benchmark
  題庫而作廢;第一輪盲評因 PS 引號傳遞/stdin 死等等四個傳輸層 bug 作廢重跑——
  全部過程見 git log 與 judge.ps1 註解。
